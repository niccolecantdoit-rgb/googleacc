# Plan: google-account-vault

目标：做一个可部署的网站，用于记录所有谷歌账号信息（包含密码），支持：登录、标签自定义与分配、筛选、拖拽排序账号、拖拽分配标签。

## 范围内字段（账号记录）
- 账号（邮箱/用户名）
- 密码
- F2A 情况：链接 / 手机（可选：未知）
- 恢复邮箱
- 恢复手机
- 验证用手机号
- 标签（多选、用户自定义）

## Work Plan (checkboxes)

- [ ] 01 Scaffold 项目：Next.js(TS) + 基础 UI + 环境变量模板 + README（本地运行/部署）
- [ ] 02 数据层：Prisma + SQLite（或同等简单方案）定义 Account/Tag/AccountTag + 迁移
- [ ] 03 鉴权：单用户登录（首次初始化设置密码；后续登录校验；会话/cookie）
- [ ] 04 敏感字段处理：账号密码等在数据库中加密存储；服务端解密仅用于展示/编辑
- [ ] 05 API：Account CRUD、Tag CRUD、分配/取消标签、拖拽排序（账号顺序、标签顺序）
- [ ] 06 前端页面：登录页 + 账号列表页（表格/卡片）+ 新增/编辑弹窗 + 标签管理
- [ ] 07 交互：
  - [ ] 07a 列表拖拽排序账号（持久化 order）
  - [ ] 07b 拖拽把账号分配到某个标签（持久化）
- [ ] 08 筛选：标签多选 + 关键字搜索 + F2A 类型 + 状态（缺字段/仅显示缺字段）
- [ ] 09 验证：lint/typecheck/build 通过；最小手动验收流程记录到 README
- [ ] 10 打包部署：提供 Dockerfile（或等价）+ 生产启动说明

## Done Definition
- 所有复选框完成
- 本地可运行（开发/生产）
- 登录可用；账号/标签增删改查可用
- 拖拽排序与拖拽打标可用
- 筛选可用
